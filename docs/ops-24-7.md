# 24/7 Operations Runbook

## Process management

Aanbevolen: PM2 voor autorestart en gecontroleerde reloads.

```bash
npm run start:pm2
npm run reload:pm2
pm2 save
```

## Monitoring checks

- Liveness: `/api/health`
- Readiness: `/api/ready`

Voor externe monitoring elke 1 minuut:
- Alert bij 2 opeenvolgende failures
- Escalatie bij >5 minuten downtime

## Dagelijkse taken

1. Backup draaien (`npm run backup:db`)
2. Logcontrole op `failed` email events
3. Health/readiness controle

## Wekelijkse taken

1. Retentie/pseudonimisering draaien:

```bash
npm run retention:pseudonymize
```

2. Review van open beveiligingsupdates (`npm audit`)

## Maandelijkse taken

1. Rapportages versturen:

```bash
npm run reports:monthly
```

2. Restore-test op testdatabase uitvoeren

## Incidentrespons (samenvatting)

1. Bevestig storing met `/api/health` en `/api/ready`
2. Check PM2 status en logs
3. Indien nodig rollback naar vorige release + `npm run deploy:cloud86`
4. Leg oorzaak en herstelactie vast in incidentlog
